# 如何访问系统？

## 🎯 快速回答

部署完成后，根据您的部署位置：

### 在本地电脑运行
打开浏览器输入：**http://localhost:3000**

### 在服务器上运行
打开浏览器输入：**http://您的服务器IP:3000**

例如，如果服务器IP是 `192.168.1.100`，则输入：
```
http://192.168.1.100:3000
```

---

## 📋 详细说明

### 1️⃣ 找到您的服务器IP地址

**方法一：在服务器上运行命令**
```bash
# 查看公网IP（如果是云服务器）
curl ifconfig.me

# 或查看所有IP地址
ip addr show
```

**方法二：在云服务商控制台查看**
- 阿里云：进入ECS实例页面，查看"公网IP"
- 腾讯云：进入云服务器列表，查看"主IP地址"  
- AWS：进入EC2实例，查看"公有IPv4地址"

### 2️⃣ 启动服务

**使用Docker（推荐）：**
```bash
cd docker
docker-compose up -d
```

**手动启动：**
```bash
# 后端
python -m uvicorn backend.api.main:app --host 0.0.0.0 --port 8000

# 前端（新终端）
cd frontend
npm run dev -- --host 0.0.0.0
```

### 3️⃣ 访问系统

在浏览器输入以下地址之一：

| 访问方式 | 前端地址 | 后端API地址 |
|---------|---------|------------|
| **本地访问** | http://localhost:3000 | http://localhost:8000 |
| **同局域网访问** | http://局域网IP:3000 | http://局域网IP:8000 |
| **远程访问** | http://公网IP:3000 | http://公网IP:8000 |

### 4️⃣ 开放防火墙（重要！）

如果无法远程访问，需要开放端口：

**Linux服务器：**
```bash
# Ubuntu/Debian
sudo ufw allow 3000
sudo ufw allow 8000

# CentOS/RHEL  
sudo firewall-cmd --add-port=3000/tcp --permanent
sudo firewall-cmd --add-port=8000/tcp --permanent
sudo firewall-cmd --reload
```

**云服务器：**
还需要在云服务商控制台的"安全组"中添加规则，允许3000和8000端口的入站访问。

## 🖥️ 界面预览

访问成功后，您将看到：

- **仪表盘**：系统概览和最近运行记录
- **论文管理**：浏览抓取的论文
- **配置中心**：调整关键词、评分规则、推送设置等
- **定时任务**：设置自动运行时间
- **日志查看**：查看系统运行日志

## ❓ 常见问题

### Q: 输入地址后无法访问？
**A:** 检查：
1. 服务是否启动：`docker ps` 或 `netstat -tlnp | grep 3000`
2. 防火墙是否开放：`sudo ufw status`
3. 云服务器安全组是否配置

### Q: 能访问但页面显示不正常？
**A:** 
1. 按F12打开浏览器开发者工具，查看Console是否有错误
2. 检查后端服务是否正常：访问 `http://<IP>:8000/health`

### Q: 只能本地访问，外网访问不了？
**A:** 
1. 确认前端启动时使用了 `--host 0.0.0.0`
2. 检查服务器防火墙规则
3. 检查云服务器安全组配置

### Q: 想使用域名访问怎么办？
**A:** 需要：
1. 有一个域名并解析到服务器IP
2. 配置Nginx反向代理
3. （可选）配置SSL证书实现HTTPS

详细配置参考：[docs/REMOTE_ACCESS.md](REMOTE_ACCESS.md)

## 🚀 快速测试

启动服务后，在命令行测试：

```bash
# 测试后端是否正常
curl http://localhost:8000/health

# 如果返回 {"status":"healthy"} 说明后端正常
```

然后在浏览器打开前端地址即可。
